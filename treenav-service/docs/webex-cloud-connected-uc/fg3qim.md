# Network Requirements for Webex Cloud-Connected UC

## **Prerequisites** 

- Ensure that you're able to log in to [Control Hub](http://admin.webex.com). For more info, see [Get Started with Cisco Webex Control Hub](https://help.webex.com/en-us/nkhozs6/).
- Bandwidth Usage: For a larger cluster, ensure that you have at least two MBPS of bandwidth.
- Network Proxy: The Cloud-Connected UC service supports both unauthenticated and authenticated HTTP proxies. The supported proxy authentication schemes are Basic and Digest.
- Enable support for WebSocket traffic.
- Cloud-Connected UC cloud port 443 must be reachable from the customer environment.
- Enable CDR and CMR in Unified CM.
- Configure the Unified CM clusters with Network Time Protocol (NTP). The NTP server must be synchronized with the international UTC clock.

## **Types of traffic** 

Data over HTTPS connections is outbound only and uses URLs for communication.

Cloud-Connected UC supports both Transport Layer Security (TLS) version 1.2 and 1.3.

## URLs required to access Cloud-Connected UC service

If your organization uses a proxy, ensure that you can access the following URLs:

**\*.ucmgmt.cisco.com** 

**\*.webex.com** 

**\*.wbx2.com** 

IP addresses are chosen from the available Cisco and Amazon Web Services (AWS) address pools.

## Firewall requirement

If a firewall is configured between UC Apps and the Webex Cloud Connect UC, SSL inspection for that traffic must be disabled.

##  Data Encryption in Transit 

Cloud-Connected UC agents validate the Cloud-Connected UC connection with a certificate (Amazon Root CA 1 certificate). This validation ensures that the certificates presented when a TLS session establishes can be validated against the trusted root CA certificates. The trusted root CA certificates are installed on the device operating system.

Cloud-Connected UC doesn't offer Online Certificate Status Protocol (OCSP) Stapling to check the validity of the SSL certificate.

Both TLS version 1.2 and 1.3 are currently supported.
