# Limitations for polls and Q&A in Slido

Slido is available in Webex Meetings and Webex Webinars. You can use Slido in Webex Meetings on [version 41.6 and later sites](https://help.webex.com/qgij81/), and in Webex Webinars on [version 41.9 and later sites](https://help.webex.com/qgij81/) and in Webex Suite meeting platform. Slido polls and Q&A in webinars are available for up to 10,000 attendees, while quizzes are available for up to 5,000 participants.

## Limitations

- Webex users must be created in or linked to a Control Hub organization.
- Slido isn’t available to meeting hosts in Site Admin managed sites, unless the site and users are linked to Control Hub and they use the same method of Webex authentication as Control Hub-managed users.
- Microsoft Edge WebView2 is required, if using Windows.  
Otherwise, Slido prompts you to install it on first use and WebView2 downloads from Microsoft. Download and install can take several minutes, which could impact your meeting experience.
- Slido isn't supported in meetings from a space in the Webex App. Slido can be launched and managed from the Meetings desktop app or from full-featured meetings in Webex App. Hosts can also go to <https://www.slido.com> and prepare and run Slido from there.
- Webex room devices and desk devices for hosts aren't supported.
- Participants can use Slido on the Meetings, Webex Webinars and Webex Suite Meeting Platform desktop, mobile and web apps.  
Slido on personal devices is only supported for participants in Meetings and Webex Suite Meeting Platform.
- If you make someone else the host during a meeting, the new host can't create and run polls and Q&A in Slido if the host already opened Slido via Apps during the meeting. The new host can request access to the active Slido as a collaborator unless they're from a different Webex organization.  
To add a collaborator: Open Settings from Slido panel and [add collaborators to help you manage Slido in Webex Meetings](https://community.slido.com/setting-up-a-slido-82/use-co-hosts-to-collaborate-and-help-manage-slidos-501?tid=501&fid=82#Allow%C2%A0external+co-hosts%C2%A0when+SSO+is+set+for+your+Slido+organization:~:text=the%20same%20license.-,Allow%C2%A0external%20co-hosts%C2%A0when%20SSO%20is%20set%20for%20your%20Slido%20organization,-If%20you%27re%20using).
- External guests need to be [enabled in Slido Organization settings](https://community.sli.do/use-slido-with-webex-90/add-collaborators-to-help-you-manage-slido-in-webex-meetings-1295). Only External Slido co-hosts with a Webex account can manage Slido polls and Q&A. They can do so from the Slido website at <https://www.slido.com>.
- The color theme in Slido doesn't match the color theme in Meetings.  
Workaround: There’s an option for hosts and meeting participants who launched Slido to [customize the colors for participants](https://community.sli.do/setting-up-a-slido-event-82/add-branding-and-custom-colors-to-your-event-454).
- Polls and Q&A are included in meeting recordings only if Slido [Present mode](https://community.sli.do/running-a-slido-event-81/display-audience-questions-voting-results-and-ideas-for-participants-in-present-mode-437) is screen-shared from a web browser or through [Google Slides](https://www.sli.do/features-google-slides) or [PowerPoint](https://www.sli.do/powerpoint-polling) integrations.
- Slido isn't supported in breakout sessions.
- The host needs to [refresh Slido after closing breakout rooms](https://community.slido.com/slido-for-webex-91/slido-disappears-for-participants-after-returning-from-breakout-rooms-in-webex-4124) in Meetings or Webex Suite Meeting Platform.
- Slido functionality isn’t available in end-to-end encrypted meetings.
- Slido isn't supported if you use the Meetings web app in Internet Explorer 11.
- Slido isn't supported in Webex Events (classic) .

See also [Additional limitations for organizations hosting data in the US datacenter](https://help.webex.com/en-us/article/nmrk74x/Integrate-Slido-with-Webex-App#Cisco%5FReference.dita%5Fb6322080-8d6c-4b0f-aef9-3ee15d6fc726).
