# Troubleshoot Users Not Receiving Emails from Webex in Control Hub

## Check the status of the last email sent to a user

When a user gets an email from Webex, you can check and see if that email was successfully delivered. The following table shows the possible statuses that an email can have, and what actions you can take to help troubleshoot if the email hasn't been delivered yet.

| Email status | Description                                                        | Action                                                                                                                                                                                                                    |
| ------------ | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| None         | No email was recently sent to the user.                            | NA.                                                                                                                                                                                                                       |
| In Progress  | The email is currently getting sent to the user.                   | No action is needed.                                                                                                                                                                                                      |
| Delivered    | The email was successfully sent to the user.                       | No action is needed.                                                                                                                                                                                                      |
| Unknown      | An email was sent to the user, but the status couldn't be tracked. | Wait for some time, and if the status doesn't change, you can try resending the email.                                                                                                                                    |
| Error        | The email wasn't successfully sent to the user's email address.    | You can see the response from the user's email address server on why they didn't receive the email. You can also clear the user's email address from the bounce (suppression) list before trying to send the email again. |

| 1 | Sign in to [Control Hub](https://admin.webex.com), go to Users, and select a user.   |
| - | ------------------------------------------------------------------------------------ |
| 2 | In the Summaries tab, scroll down to the Info section to view the Last email status. |

## Remove a user's email address from the Bounce (Suppression) List

If users aren't receiving emails from Webex services, then that could mean that their email addresses are on Cisco's suppression list. If the last email sent to them has an error status, then a full administrator can clear those users from the suppression list.

| 1 | Sign in to [Control Hub](https://admin.webex.com), go to Users, and select a user.   |
| - | ------------------------------------------------------------------------------------ |
| 2 | In the Summaries tab, scroll down to the Info section to view the Last email status. |
| 3 | Click Remove from Bounce List.                                                       |
