# Cisco managed third-party device support on Webex Calling

## Onboard third party devices

This feature helps migrating customers who have already made a significant investment in third party devices to onboard to Webex Calling.

### Supported models:

See the [Supported devices for Webex Calling](https://help.webex.com/en-US/article/qkwt4j/Supported-devices-for-Webex-Calling).

### Device capabilities: 

- Provisioning and deployment

  - Assign to users or workspaces
  - Support primary and shared lines
  - Support monitored lines
  - Support standard sip phone features (no advanced feature support)
  - Manual deployment (no activation code or EDOS support)
  - Individual device provisioning in Control Hub (no bulk)
  - Locked device configuration (no third-party device management)
  - Manage device settings in Control Hub

### Provisioning and deployment:

**Provisioning from Control Hub:**

Once the devices are exposed to the customer, you can provision third party devices with MAC address as managed devices through Control Hub. See [Configure and manage devices in Control Hub](https://help.webex.com/en-US/article/n9r1aac/Configure-and-manage-Webex-Calling-devices).

Third party devices appear in the existing Device list along with the other Cisco devices.

For manual provisioning and deployment, see sections [Manually deploy AudioCodes for 4xxHD devices](#task-template%5F581813e7-60ca-49b3-8d01-bf9a53fe41f6), [Manually deploy Poly devices](#task-template%5Fce572287-cbe4-4bfe-9129-81e820f2ac77), and [Manually deploy Yealink devices](#task-template%5F38ae094c-1f90-4ce6-8d79-682e189f7e8e).

### Device information and analytics

Device information or analytics:

- Third party devices don’t appear in the Control Hub analytics dashboards and no call quality metrics are available.
- Third party devices appear in the Devices tab of Control Hub and their statuses are displayed.

### Technical Support

- Supported through the normal TAC ticketing process
- Basic SIP troubleshooting including SIP registration verification and basic calling capabilities. No remote log collection or customer syslog capability.
- No third-party feature support
- No third-party RMA or warranty support

Use these steps to deploy AudioCodes for 4xxHD device manually:

| 1 | Reset the phone to factory default settings. You can use one of these methods: If the user knows the administrator password, use this procedure: Press the Menu key on the device and select the Admin option shown on the screen. The device prompts the user to enter the admin password. By default, the admin password is 1234. Select the 4th option Restore Defaults. Then, select Yes or No to restore the device to factory default settings. If the user does not know the administrator password, use this procedure: Press and hold the Menu and OK keys on the device simultaneously. If POE is used by the device, unplug the power cable or ethernet cable. Plug the power or ethernet cable back into the device while continuing to hold the Menu and OK keys on the device. Hold these keys for 5 seconds. Once released, the device is restored to the default settings. |

Use the following steps to deploy a Yealink device manually.

| 1 | Reset the phone to factory default settings: Press and hold the OK key on the face of the phone for 10 seconds. This prompts you to accept or decline a factory reset. If you accept, the unit restores itself to factory default settings without the need for the admin password or logging in. Begin from the home screen of the phone for this procedure to work. If there are other alerts present, such as a missed call notification, clear the notification to proceed to the home screen.                                                              |
| - | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 2 | To manually update the device configuration through the phone menu, go to Menu \>  Status.                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| 3 | Enter the IP address into your browser, when prompted for login.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 4 | Log in to the device using the default (case-sensitive) username ("admin") and password ("admin"), and then click Confirm.                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| 5 | Under the Settings tab, click on Auto Provision.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 6 | Enter the provisioning server address into the Server URL field: US: <https://yealink.sipflash.com> UK: <https://yealink-uk.bcld.webex.com/> EU: <https://yealink.broadcloud.eu> EUN: <https://yealink-eun.bcld.webex.com> AU: <https://yealink.broadcloud.com.au> CA: <https://yealink-ca.bcld.webex.com> JP: <https://yealink-jp.bcld.webex.com> KSA: <https://yealink-ksa.bcld.webex.com> India: <https://yealink-in.bcld.webex.com>                                                                                                                         |
| 7 | Click Confirm.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| 8 | Click Auto Provision Now. This causes the device to request the files from the provisioning server and reboot. If the station assignment process was properly completed before the reboot, the device boots into a station specific assigned/usable state. If not, the device boots into a default unassigned/unusable state and remains in this state until the site administrator completes the station assignment process. If this occurs, contact your site administrator, and request for completion of the station assignment for the device in question. |
